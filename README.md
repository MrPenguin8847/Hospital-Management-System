# Dhami Hospital — Hospital Management System

A complete, locally runnable **Hospital Management System** built with **HTML5**, **CSS3**, **vanilla JavaScript**, **PHP**, and **MySQL**. The interface uses a **dark, futuristic** aesthetic: **glassmorphism**, **neon blue/purple gradients**, **smooth hover states**, and **responsive** dashboard layouts.

---

## Project overview

Dhami Hospital provides:

- A **public landing page** with hero, services, about, and a **contact form** stored in the database.
- **Session-based authentication** with **password hashing** (`password_hash` / `password_verify`).
- An **admin panel** for operations, billing, reports, tests, complaints, and analytics.
- A **patient portal** for profile management, appointments, medical PDFs, bills, and complaints.
- **Prepared statements** throughout and basic **input sanitization** for safer handling of user data.

---

## Features

### Public

- Futuristic landing page (hero, about, services, contact form, **Book Appointment** CTA).
- Patient self-registration.
- Dual login (admin / patient).

### Admin

- Dashboard with **stats** (patients, appointments, revenue) and **CSS bar charts**.
- **Profile** management (name, email, password).
- **Add new admin** accounts.
- **Register patients** from the panel; **view/delete** patients.
- **Approve / reject** appointment requests.
- **Create bills**, **record payments**, **printable HTML invoice** (browser print / save as PDF).
- **Upload PDF medical reports** per patient; **delete** reports.
- **Medical tests** catalog (CRUD).
- **Complaints** inbox with **resolve** workflow and admin response.
- **Analytics** page (revenue snapshot, status mix, department volume).
- **Contact inbox** for messages submitted from the landing page.

### Patient

- Dashboard **overview** cards.
- **Profile** editor (optional password change).
- **Book appointments** (department, doctor, date/time) — **pending** until admin approves.
- **Appointment history** with status.
- **Medical reports** list with **secure PDF download** (owner-only).
- **Bills** list and **invoice** view.
- **Complaints** — submit and track resolution.

### Security notes (read before production)

- Configure MySQL credentials in `includes/config.php`.
- After first deployment, **delete or protect** `setup.php`.
- Ensure the `uploads/reports/` directory is **not executable** (see `uploads/reports/.htaccess` for Apache).
- Use **HTTPS** in production; tighten cookie flags as appropriate.

## Installation (XAMPP / WAMP)

### 1. Place the project

Copy the `hospital-management-system` folder into your web root, for example:

- **XAMPP (Windows):** `C:\xampp\htdocs\hospital-management-system`
- **WAMP:** `C:\wamp64\www\hospital-management-system`

### 2. Start services

- Start **Apache** and **MySQL** from the XAMPP/WAMP control panel.

### 3. Create the database

**Option A — phpMyAdmin**

1. Open `http://localhost/phpmyadmin`
2. Click **Import**
3. Choose `database/schema.sql`
4. Click **Go**

**Option B — MySQL CLI**

```bash
mysql -u root -p < database/schema.sql
```

### 4. Configure database credentials

Edit `includes/config.php`:

```php
define('DB_HOST', 'localhost');
define('DB_NAME', 'hospital_management');
define('DB_USER', 'root');
define('DB_PASS', ''); // your MySQL password
```

### 5. First-time admin account

The schema does **not** store a plaintext password. After importing the database, open in your browser:

`http://localhost/hospital-management-system/setup.php`

This creates the default administrator:

| Field    | Value        |
|----------|--------------|
| Username | `admin`      |
| Password | `Admin@123`  |

**Remove or restrict `setup.php` after use.**

Alternatively, insert an admin manually using a bcrypt hash generated by PHP (`password_hash`).

### 6. Uploads directory

Ensure PHP can write to:

`uploads/reports/`

On Windows/XAMPP this usually works after the first upload; if not, create the folder manually and grant write permission to the web server user.

### 7. Access the app

- **Landing:** `http://localhost/hospital-management-system/`
- **Admin login:** `http://localhost/hospital-management-system/login.php?role=admin`
- **Patient login:** `http://localhost/hospital-management-system/login.php?role=patient`

Register a patient via **Register** or create one from the admin **Patients** screen.
